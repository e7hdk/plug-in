Act as the `product` agent. Idea: $ARGUMENTS

1. Clarify the problem, the target user, and the success metric.
2. Write a concise spec to `docs/specs/` (problem, scope, out-of-scope, acceptance criteria, risks).
3. Open a GitHub Issue per deliverable: acceptance-criteria checklist, `agent:` owner label, `state:spec`.

Return the issue numbers.
